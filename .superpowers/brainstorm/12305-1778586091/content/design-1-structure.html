📊 数据加载 (data/loader.py)

核心职责：
  • 启动时一次性加载 metric_data.csv 到 Pandas DataFrame
  • 数据预处理：添加 date 列 (year-month) 便于时间轴绘图
  • 构建元数据：提取科室列表、指标列表（含编码和单位），供 System Prompt 使用
  • 使用 @st.cache_data 缓存，避免每次交互重新加载

🔗 因果关系图 (data/causal_graph.py)

核心职责：
  • 加载 causal_relations.xlsx 构建 NetworkX 有向图
  • 边：因子指标 → 结果指标 (属性: category=类别)
  • 节点：指标名称 (属性: code=编码)
  • 提供 get_factors(metric_name) → 返回 {类别: [因子列表]} 字典
  • 提供 get_drilldown(metric_name) → 查找可下钻的因子（既是因子又是结果指标的）
多层嵌套处理：
5个指标既是因子又是结果：门诊人次、门急诊人次、出院人次、出院患者手术人次、每出诊单元门诊人次
→ get_drilldown() 标识这些指标，在分析结果中提示"可进一步下钻"
